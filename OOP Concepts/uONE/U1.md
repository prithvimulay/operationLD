# Objects 
- these are real world entity like a car, a person, a bank account, etc.
- anything that can be described as a thing with characteristics (state) and capabilities (behavior) is an object.
Characteristics are also known as attributes/properties/state.
eg: Car is a object having characteristics like color, brand, model, etc.
Behaviors are known as methods/actions.
eg: Car is a object having behaviors like start(), stop(), brake(), drive() etc.

Objects have three main characteristics:

1. State/Attributes/Properties: The data or attributes of the object. In the Person class, the name and age are the object's state.
2. Behavior/Methods/Functions: The methods and functions that the object can perform. In the Person class, the display method is mentioned here as an example of behavior, and it will be implemented later in this lesson.
3. Identity: A unique identifier distinguishes the object from others, even if they have the same state. This is handled by the memory address in JavaScript.

An object is thus a concrete instance of a class that includes state, behavior, and identity.

# Enums
* An enum (short for enumeration) is a special data type that defines a fixed set of named constants. 
* Unlike strings or integers, enums are type-safe, meaning the compiler ensures you can only use values that actually exist in your defined set.
* They ensure that a variable can only take one out of a predefined set of valid options.
    ```
    const OrderStatus {
        PLACED = "PLACED",
        CONFIRMED = "CONFIRMED",
        SHIPPED = "SHIPPED",
        DELIVERED = "DELIVERED",
        CANCELLED = "CANCELLED"
    }
    ```

* Without enums, you'd represent order statuses as strings scattered throughout your codebase:
    ```
    string status = "Pending";

    if(status === "pingding"){
        .....
    }
        <!-- we made a typo -->
    ```
* This code compiles without any warnings. The typo "PNDING" is a perfectly valid string as far as the compiler is concerned. The bug only surfaces when a customer complains that their order never gets processed.

* when we use enums the compiler knows exactly what values are valid

* so no more scattered strings or magic numbers (1,2,3...) in the code

* no guess work - OrderStatus.SHIPPED >>>> OrderStatus.3

* eg:
    1. User Roles (e.g., ADMIN, CUSTOMER, DRIVER)
    2. Vehicle Types (e.g., CAR, BIKE, TRUCK)
    3. Directions (e.g., NORTH, SOUTH, EAST, WEST)

* Enum value can hold additional data and even define behavior. 
    ```
    const Coin = Object.freeze({
        PENNY: 1,
        NICKEL: 5,
        DIME: 10,
        QUARTER: 25
    });

    // Usage
    console.log(Coin.DIME); // Outputs: 10

    <!-- Object.freeze makes the code immutable - so no one can change it like - Coin.DIME = 1000 -->
    ```
